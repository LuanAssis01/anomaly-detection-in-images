LABORATÓRIO DE DETECÇÃO DE ANOMALIAS EM IMAGENS - TCC
================================================================================

🎯 OBJETIVO
-----------
Comparar diferentes arquiteturas de deep learning (CNNs e Transformers) para 
detecção de falsificações em imagens, gerando dados experimentais para artigo 
científico de TCC.

📊 MODELOS IMPLEMENTADOS
-------------------------
1. ResNet-50       - CNN clássica pré-treinada (baseline)
2. ResNet-101      - CNN mais profunda
3. ViT             - Vision Transformer (google/vit-base-patch16-224)
4. CvT-13          - Convolutional Vision Transformer (leve)
5. CvT-21          - Convolutional Vision Transformer (médio)
6. CvT-W24         - Convolutional Vision Transformer (pesado)
7. DINOv2          - Self-supervised Vision Transformer

🚀 INÍCIO RÁPIDO
----------------

1. INSTALAR DEPENDÊNCIAS:
   pip install -r requirements.txt

2. TESTAR AMBIENTE:
   python test.py

3. TREINAR MODELOS:
   
   # Todos os modelos (recomendado para TCC):
   python src/train.py --model all --epochs 50
   
   # Modelo específico:
   python src/train.py --model cvt21 --epochs 50
   
   # Opções disponíveis:
   --model resnet50      # ResNet-50
   --model resnet101     # ResNet-101
   --model vit           # Vision Transformer
   --model cvt13         # CvT-13 (leve)
   --model cvt21         # CvT-21 (médio, recomendado)
   --model cvt_w24       # CvT-W24 (pesado)
   --model dinov2        # DINOv2

4. AVALIAR MODELOS:
   python src/evaluate.py --model all --visualize

5. ANÁLISE COMPARATIVA:
   python src/analysis_notebook.py

6. DEMO INTERATIVA:
   python src/app.py

🔬 METODOLOGIA PARA O TCC
--------------------------
1. COLETA DE DADOS
   - Dataset balanceado de imagens autênticas e falsificadas
   - Máscaras de segmentação para regiões manipuladas
   - Split: 80% treino, 20% validação

2. TREINAMENTO
   - Data augmentation: flip, rotação, brilho, contraste
   - Otimizador: Adam (lr=1e-4)
   - Loss: CrossEntropyLoss
   - Early stopping baseado em validation accuracy
   - Batch size: 16 (ajustável conforme GPU)

3. AVALIAÇÃO
   - Métricas em conjunto de validação
   - Análise de erros (FP, FN)
   - Comparação estatística entre modelos
   - Visualização de predições

4. ANÁLISE COMPARATIVA
   - Curvas de aprendizado
   - Tempo de treinamento
   - Número de parâmetros
   - Trade-off acurácia vs eficiência
   - Análise qualitativa das predições

🎓 HIPÓTESES DO TCC
-------------------
H1: Transformers (ViT, CvT, DINOv2) superam CNNs tradicionais na detecção 
    de anomalias sutis em imagens
    
H2: CvT oferece melhor trade-off entre acurácia e eficiência computacional 
    ao combinar convoluções com self-attention
    
H3: DINOv2, com aprendizado auto-supervisionado, generaliza melhor com 
    poucos dados rotulados
    
H4: Maior número de parâmetros nem sempre resulta em melhor desempenho 
    (CvT-W24 vs CvT-21)

🛠️ CONFIGURAÇÕES AJUSTÁVEIS
----------------------------
Edite configs/config.py para alterar:
- BATCH_SIZE: 16 (reduzir se GPU tiver pouca memória)
- NUM_EPOCHS: 50
- LEARNING_RATE: 1e-4
- IMAGE_SIZE: 224
- DEVICE: 'cuda' ou 'cpu'

⚠️ REQUISITOS COMPUTACIONAIS
-----------------------------
- GPU: Recomendado 8GB+ VRAM (RTX 3070 ou superior)
- RAM: 16GB+
- Espaço: ~10GB para modelos e resultados
- Tempo: ~2-4 horas por modelo (dependendo do hardware)

🆘 TROUBLESHOOTING
------------------
- CUDA out of memory: Reduza BATCH_SIZE em config.py
- Modelos não carregam: Execute python test.py
- Erro de path: Certifique-se de estar no diretório raiz
- Dataset vazio: Verifique se data/train_images existe
